
Goal: make the Information step open to the teaching-choice menu first, stop auto-switching into long text, keep the lesson as a true full-page scroll view on iPad/mobile, and start TJ audio when the student chooses a teaching mode.

What’s actually going wrong
- The Information step still auto-fetches `expandedInfo` as soon as the step opens (`LearningOrbDialog.tsx`), so the first view briefly appears and then immediately turns into the long text view from the second screenshot.
- The long explanation is being generated by the wrong path for this experience, and it is leaking stray metadata-like text (`{"voice_id": ...}`) into the lesson.
- The lesson is inside a Radix dialog whose base content uses modal/grid styling; on iPad Safari this can still cause nested scroll trapping even though the inner area has `overflow-y-auto`.
- Auto-read is tied to async loading, which is less reliable on mobile Safari than speech triggered from the student’s tap.

Plan
1. Make the Information step menu-first and opt-in
- File: `src/components/LearningOrbDialog.tsx`
- Remove the automatic `fetchExpandedInfo()` trigger for the Information step.
- Remove the automatic loading state that pushes the menu down.
- Render the Information step as:
  - short intro/context at the top
  - “Choose How TJ Teaches This” menu immediately below
  - generated content only after the student taps a mode

2. Stop mixing two competing explanation systems
- Files: `src/components/LearningOrbDialog.tsx`, `src/components/TJLearningStudio.tsx`
- Retire the separate `expandedInfo` panel from the Information step.
- Let `TJLearningStudio` become the single source for summary, guided explanation, slideshow, teaching flow, and explain-again content in that section.
- This keeps the menu at the top and prevents the screen from jumping to a different renderer.

3. Clean the generated content before display/speech
- Files: `src/components/TJLearningStudio.tsx` and, if needed, `supabase/functions/ai-mentor-chat/index.ts`
- Add a sanitize step that strips code fences, leading JSON/object fragments, and voice/tool metadata before rendering or reading aloud.
- If the noisy text is coming from the backend prompt, tighten that prompt so lesson text returns plain teaching copy only.

4. Fix the full-screen dialog shell for real scrolling
- Files: `src/components/ui/dialog.tsx`, `src/components/LearningOrbDialog.tsx`
- Add a proper fullscreen dialog variant instead of inheriting the default centered modal/grid behavior.
- Use a strict layout:
  - fixed fullscreen shell
  - non-scrolling header/progress area
  - one scrollable center reading area
  - `min-h-0`, `overflow-hidden` on the shell
  - `overflow-y-auto`, `overscroll-contain`, `touch-pan-y`, and `-webkit-overflow-scrolling: touch` on the reading area
- This should resolve the iPad/Safari “content visible but won’t scroll” behavior.

5. Make generated teaching content scroll in the main page, not inside itself
- File: `src/components/TJLearningStudio.tsx`
- Keep the menu above the content.
- Ensure explanation/slideshow/teach-flow cards do not create their own vertical scroll areas.
- Let all long content expand naturally inside the dialog’s single main scroll container.

6. Make audio start from the student’s choice
- Files: `src/components/TJLearningStudio.tsx`, `src/components/LearningOrbDialog.tsx`
- When the student taps:
  - Guided Explanation / Summary / Explain Again: auto-read the cleaned result immediately
  - Slideshow: auto-read slide 1 body + speaker note
  - Teaching Flow: auto-read step 1
  - Audio Script: auto-read the script immediately
- Because playback starts from a tap, it will be much more reliable on iPad/mobile than async autoplay after background loading.

Files to modify
- `src/components/LearningOrbDialog.tsx`
- `src/components/TJLearningStudio.tsx`
- `src/components/ui/dialog.tsx`
- Possibly `supabase/functions/ai-mentor-chat/index.ts` if the metadata leak is confirmed to originate there

Expected result
- The student lands on the teaching menu first and it stays above the content.
- The screen no longer auto-switches into the long explanation on its own.
- Long lessons scroll smoothly up and down while TJ is speaking.
- The random `voice_id` text disappears.
- The Information step feels like one spacious reading experience instead of a trapped modal-within-a-modal.
